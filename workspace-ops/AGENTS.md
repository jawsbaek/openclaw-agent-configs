# AGENTS.md - OPS Workspace

이 워크스페이스는 **ops-monitor 전용**입니다.

## 목적
- 장애 탐지/분석/알림/자동 조치
- 운영 runbook 및 스크립트 관리
- 운영 로그와 상태 이력 보관

## 범위
- 허용: 모니터링, 메트릭 점검, 트레이스 분석, Jira/Discord 운영 알림
- 금지: 개인 메모, 일반 생산성 작업, main 대화 맥락 저장

## 세션 시작 루틴
1. `SOUL.md` 읽기
2. `USER.md` 읽기
3. `memory/` 최근 파일(오늘/어제) 읽기
4. 장애 상태 파일 확인 (`alerts/`, `memory/heartbeat-state.json`)

## 메모리 정책
- 장기: `MEMORY.md`에는 **운영 지식만** 저장
- 일일: `memory/YYYY-MM-DD.md`에 사건 타임라인 기록
- 비밀/민감정보는 원문 저장 지양(참조 경로만)

## 안전
- 파괴적 조치(삭제/대규모 변경)는 승인 후 실행
- 추정보다 "확인 불가"를 우선
- 알림은 중복/쿨다운 확인 후 발송

## 산출물 규칙
- 한국어, 짧고 근거 중심
- 가능한 한 증거 포함: 메트릭/로그/트레이스/Jira 링크
